# GEOCON Atlas — Uzman Audit & Multi-Taxon Vertical Blueprint

2026-06-02 · sonu sabaha denk gelen otonom oturumun ardından kapsamlı
bir gözden geçirme. **Çalıştırılmadı, plan olarak duruyor.**

Not: bu doküman ürün-vizyonu-eleştirisi değil; **architecture +

## I. Genel kanı — tek paragraf

Sistem teknik olarak güçlü ama **bilgi mimarisi (IA) over-engineered**.
**15 sidebar item + 12 home widget + 3-katmanlı footer + Spotlight
ribbon + breadcrumb + bell + theme switch + arama butonu** — bir
araştırmacı `/geocon`'a ilk girdiğinde **yaklaşık 30 ayrı interactive
yüzeyle** karşılaşıyor. Hiçbir araştırma platformu (bbioRxiv, GBIF,
IUCN Red List, OpenAlex, ORCID) ilk ekranda bu kadar yüzey
göstermiyor. Birinci öncelik **information density'i yarıya
indirmek**, ikinci öncelik **navigation'ı 3 bucket'a oturtmak**.

## II. Sidebar — kritik sorun (haklı tespitin)

### Şu anki durum (sayım)

15 item + (admin ise) Admin. Hepsi flat list, hepsi aynı boyut, aynı
indent. Mental model'ler karışmış:

| Item | Tip | Bucket |
|---|---|---|
| Home | gateway | (root) |
| Ask GEOCON | tool | tools |
| Compare | tool | tools |
| Field log | tool | tools |
| Activity | stream | workspace |
| Explore | tool | tools |
| Programs | actor-entity | workspace |
| ATLAS (Species) | data-entity | commons |
| Metabolites | data-entity | commons |
| Publications | data-entity | commons |
| Researchers | actor-entity | commons |
| Organizations | actor-entity | commons |
| Proposals | actor-entity | workspace |
| Open Briefs | actor-entity | workspace |
| Communities | actor-entity | workspace |

İki ayrı zihin haritası tek listeye karışmış:
- **Verbs/tools** (Ask, Compare, Explore, Field log) — "ne yapmak istiyorum"
- **Nouns/entities** (Species, Researchers, Programs) — "neyi araştırmak istiyorum"

Bu, 1990'ların Yahoo directory'sini andırıyor. Modern araştırma
platformları **2-katmanlı**: üstte 3-5 ana bucket, içinde detay.

### Önerilen reorganizasyon — 3 bucket

```
WORKSPACE (Do — kişisel iş alanı)
  ⌂ Home
  ⚡ Activity                  [badge: recent 24h]
  📋 My Programs               [badge: my count]
  📬 Proposals                 [badge: inbound]
  🗂 Open Briefs

COMMONS (Browse — paylaşılan veri)
  🌿 Species                   (Atlas)
  🧪 Metabolites
  📚 Publications
  👤 Researchers
  🏢 Organizations

TOOLS (Find — keşif araçları)
  ✨ Ask GEOCON
  ⇄  Compare
  🌍 Explore (geo/family)
  📍 Field log
```

**Bucket'lar arası**: 12px gap + overline section header
(`WORKSPACE / COMMONS / TOOLS`).

**Communities** bucket'a oturmuyor — silmek, "Researchers
> Communities" sub-tab'ına taşımak veya tamamen ertelemek opsiyonları
var. Şu an dolu mu? Bakmaya değer — boşsa siler kapatırız.

**Admin** — kendi başına, ayrı bir 4. bucket veya bottom user-pill'in
üstüne küçük "⚙ Admin" link olarak.

**⌘K zaten header'da** — Ask GEOCON'un primary nav'da yeri var mı,
düşünmeye değer. Belki Ask sadece bir sub-page, primary search ⌘K
üzerinden gider. Şu an iki paralel "araştır" yolu var, kafa karıştırıcı.

### Visual fixes (sidebar)

1. **Emoji → line icons**: Karma emoji set ("👨‍🔬" boyutu "⚡" boyutuyla
   eşit değil, tone-of-voice tutarsız, Crimson Pro tipografisiyle uyumsuz).
   `lucide-react` veya `heroicons` ile mono-line set. Senin "ciddi olmali,
   celebration kokmamali" direktifinin doğrudan ihlali.
2. **Active state**: violet 2-px border iyi ama **inactive items çok soluk**
   (`var(--gx-ink-muted)` ≈ #888). Tip okunmuyor. `--gx-ink-soft` (~#444)
   daha iyi.
3. **Item gap**: `gap: 1px` tıkış tıkış. `gap: 4px` + bucket arası 12px.
4. **"Supabase connected" Dot panel** — 9pt font, kullanıcıya hiçbir
   şey söylemiyor ama sidebar'ın değerli pixel'lerini yiyor. Sil veya
   admin debug pane'e taşı.
5. **"GEOCON v3.1" caption** — versiyon numarası araştırmacıyla
   konuşmuyor. Marketing/onboarding bağlamı dışında zarar.
6. **Logo confusion**: Sidebar'da "ATLAS" görünüyor, footer'da "GEOCON
   Atlas", URL'de `/geocon`, alt-caption'da `GEOCON v3.1`. **Ürün adı
   nedir?** Marketing brief gerekiyor: "GEOCON Atlas" tek kanonik form
   olmalı, herkes ona referans vermeli. Sidebar logo'da da öyle.

## III. Home page — 12 widget yığını

`HomeRoute.jsx` şu an 12 widget mount ediyor:

```
orcid-connect-banner  |  Conditional, sıfır mission/orcid set ise
my-mission-feed       |  Conditional, mission tag'leri varsa
trust-strip           |  4 sayı — species/threat/researcher/program
onboarding-checklist  |  Conditional, onboarding bitmediyse
spotlight-ribbon      |  "Bu hafta öne çıkan" şeritlik kart
trending-threads      |  Trending tartışmalar
quick-tools           |  Hızlı eylem butonları
my-dashboard          |  Kişisel metrics
my-atlas-history      |  Geçmiş arandı/baktım listesi
leaderboard           |  Top contributors
trending-panel        |  Trending content (üst trending-threads ile çakışıyor)
geocon-home           |  Eski monolithic home content (modüller + species grid)
```

### Sorunlar

1. **Çift trending**: TrendingThreads + TrendingPanel — aynı sinyali
   iki farklı widget gösteriyor. Birleşmeli veya biri silinmeli.
2. **Sonsuz scroll**: Hiçbir widget collapsed default değil. Bir
   araştırmacı `/geocon` açıyor, scroll'a basıyor, 10. widget'a
   gelinceye kadar bir gerçek species görmüyor.
3. **Conditional widget'lar yığına dahil edilmiyor**: orcid-connect +
   mission-feed + onboarding-checklist hepsi kapanabilir ama hangi
   sırayla kapandıkça home değişiyor — predictability zayıf.
4. **`geocon-home` (eski monolith)**: Asıl içerik (species ızgarası,
   modüller) **en alt**ta. Bu yanlış sıralanmış. Trust-strip'in
   hemen altında olmalı.
5. **Quick-tools** — sidebar'la birebir overlap. Sidebar zaten 15 yol
   sunuyor; home'da tekrar göstermek ya farklı, ya yedek olmalı.

### Önerilen Home sıralama

```
1. (cond) OrcidConnectBanner          ← only if not connected
2. (cond) OnboardingChecklist         ← only if not complete
3. (cond) MyMissionFeed               ← only if mission_tags set
4. TrustStrip                         ← 4 sayı, marketing band
5. GEOCONHome (species grid + modules)← ASIL İÇERİK, hemen yukarı
6. SpotlightRibbon                    ← editorial pick
7. TrendingPanel + TrendingThreads    ← BİRLEŞTİR
8. MyDashboard + MyAtlasHistory       ← BİRLEŞTİR ("Your activity" combined)
9. LeaderboardPanel compact
10. (sil) QuickTools
```

Net etki: 12 widget → 7 surface, asıl içerik 4. sırada, scroll yükü
yarılanır.

## IV. Visual gap'ler — ciddi kalitenin kapsadığı eksiklikler

### 1. Color accent sistemi cohesion eksik
Mevcut tokens: `--gx-accent-violet`, `--gx-accent-azure`,
`--gx-accent-bio-green`, `--gx-success`, `--gx-warning`, `--gx-danger`,
`--gx-info`. Sidebar active = violet, mobile nav active = violet,
ama Spotlight'ta azure, MyMissionFeed başlığında violet, leaderboard'da
karışık. **Tek bir "primary action" rengi seçilmemiş.** Tüm interactive
focal point'ler tek renkte (örn. violet) olmalı; semantik renkler
sadece state için (success/danger/warning).

### 2. Tipografi — Crimson Pro çok kullanılmış / değersizleşmiş
Crimson Pro (display/serif) **her başlıkta** görünüyor: sidebar,
home, species, programs… Display font'u "öne çıkan moment"lara
saklamak gerekir. Şu an her overline italic Crimson serif → görsel
hierarchy düz. Önerim:
- Display = sadece /geocon Home H1 ("ATLAS intelligence") ve hero block'lar
- H2/H3 = Inter semibold
- Italic Crimson = sadece species names (botanik konvansiyon doğru ✓)

### 3. Spacing inconsistency
Padding'ler kontrolden çıkmış. Örn:
- Shell main: `16px 20px 28px`
- Home widget container: yok, her widget kendi padding'i
- Section card padding'leri 14, 18, 20, 22 hep farklı
**Sonuç**: Sayfa "ritm"siz görünüyor. Spacing token (`--gx-sp-1` = 4px,
`--gx-sp-2` = 8px, `--gx-sp-3` = 12px, `--gx-sp-4` = 16px, `--gx-sp-5` =
24px) tanımlanmalı, herkes onu kullanmalı.

### 4. Card design fragmenti
- Bazı kartlar `border-radius: 8`, bazıları `10`, bazıları `12`, biri
  `14`. Tek değer (önerim 10 veya 12) standartlaşmalı.
- Border colors: `#ece9e2`, `#e8e6e1`, `var(--gx-border-soft)`,
  `var(--gx-border)` — ikisinden birinde karar verilmeli.
- Shadow: kimi kartta `0 10px 30px rgba(0,0,0,0.08)`, kimisinde hiç,
  kimisinde `0 1px 2px`.

### 5. Dark mode coverage eksik
ThemeSwitch var ama bazı yüzeyler dark'ta kırık — özellikle inline
`background: "#fff"` kullanan kartlar. Bu hala iki yerde:
- AdminRoute sign-in fallback: `background: "#fff"`
- PendingRow applicant note: `background: "#fff"`
- NotificationBell dropdown: `background: "#fff"`
Bunlar var(--gx-surface)'a geçirilmeli.

### 6. Empty-state stratejisi yarım
`EmptyState` shared component var ama her route kullanmıyor. Sidebar
kategorisi açıp 0 sonuç gören kullanıcı kuru "no results" görüyor.
EmptyState'i her index route'a propogate et.

## V. Functional gap'ler — eksik özellikler

### 1. Watchlist / Saved species yok
Bir araştırmacı bir species görüyor, **"Bunu izle/yer işareti"**
diyemiyor. Modern platformların temel davranışı. Tablo: `species_watch
(user_id, species_id, added_at, note)`. Sidebar'da "Watching" entry'si
(/geocon/watch).

### 2. Drafts surface'i yok
Proposal/Program/Brief yazmaya başladım, kaydetmedim → bulamam. 
Draft'lar ne profile'da listeleniyor ne sidebar'da. Tablo: zaten
`proposal_drafts` ve `program_drafts` türevleri var; UI yok.

### 3. My research / My contributions rollup yok
Bir kullanıcı kendi katkılarını (yorum + endorse + co-author +
species edit proposal + program member) **tek yerde göremiyor**.
Profile route var ama generic; "My contributions feed" yok.

### 4. Cross-entity arama yok
⌘K Spotlight var ama o sadece global navigate; içerik içinde arama
(örn. "Programs'ta status=active olanları getir") yok. Filter UI her
route'ta tekrar inşa edilmiş, paylaşılan FilterBar primitive yok.

### 5. Outcome discovery surface'i yok
L5 endorsement geldi ama "bu hafta hangi outcome'lar onaylandı"
keşif yüzeyi yok. /geocon/outcomes route'u yok. Commercialization
recognition publicly görünür olmalı.

### 6. Notification grouping yok
Bell dropdown'da 50 satır listeleniyor; "5 yeni mention", "3 TIC
assignment" şeklinde gruplanmıyor. Yüksek aktiviteli proje'de bell
kullanılmaz hale gelir.

### 7. Activity feed taban filtresi yok
/geocon/activity route'u var ama "sadece izlediğim species/program"
filter'ı yok. Watchlist + activity feed entegre edilmeli.

### 8. Onboarding feedback loop yok
Welcome adımları gönderildi → mission_tags set edildi → ama "Senin
için 12 yeni brief var" konfirmasyonu yok. MissionFeed widget var
ama o passive surface. Active follow-up email yok.

## VI. Multi-taxon vertical blueprint

Bu senin sorusunun ana parçası. Architecture sketch, **uygulama
sonra** olacak.

### Tasarım prensibi

GEOCON'un asıl tezi geofitler'e özgü değil:
- L1 Species commons + edit proposals → her taksonda çalışır
- L2 Programs (Discover→Validate→Develop→Steward) → her taksonda
  çalışır (raptor breeding programı, orkide reintroduction, mantar
  kolonizasyon hepsi aynı modüllerde)
- L3 Studies → her taksonda
- L4 Open Briefs → her taksonda
- L5 Commercialization recognition → her taksonda (etik sınırlarla)

Geophyte-spesifik olan **çok az**: `species.geophyte_type` tek
gerçek geophyte field, geri kalan 60 column generic. Bu çok iyi haber:
abstraction maliyeti düşük.

### Önerilen şema değişikliği

#### Yeni: `verticals` tablo

```sql
CREATE TABLE verticals (
  id text PRIMARY KEY,         -- 'geophytes', 'orchids', 'carnivorous_plants', 'amphibians'
  slug text UNIQUE,            -- url slug
  display_name text,           -- "Geophytes"
  description text,
  taxonomic_scope jsonb,       -- { kingdom: 'Plantae', orders: ['Asparagales','Liliales'] }
  is_active boolean,
  is_public boolean,           -- some verticals private/beta
  maintainer_org_id uuid,      -- which org curates this vertical
  brand_color text,            -- vertical-specific accent
  taxonomy_attributes jsonb,   -- which species fields are required/optional/N-A in this vertical
  created_at timestamptz,
  -- ...
);
```

#### `species` migration

```sql
ALTER TABLE species ADD COLUMN vertical_id text DEFAULT 'geophytes' REFERENCES verticals(id);
ALTER TABLE species ADD COLUMN vertical_attrs jsonb DEFAULT '{}'::jsonb;
-- Migrate existing geophyte-only fields into the jsonb bag:
UPDATE species SET vertical_attrs = jsonb_build_object('geophyte_type', geophyte_type)
WHERE geophyte_type IS NOT NULL;
-- Keep geophyte_type column for now (zero-downtime), drop after UI catches up.
```

#### Tekrarlanabilir alanlar zaten taxon-agnostic

Bunlara hiç dokunma:
- `accepted_name`, `family`, `genus` — generic taxonomy
- `iucn_status`, `cites_appendix`, `population_trend` — IUCN/CITES, evrensel
- `native_countries[]`, `introduced_countries[]`, `endemic`
- `composite_score` ve alt boyut score'lar — generic metrik
- `geocon_module` — her taksonda anlam taşır
- `photo_url`, `thumbnail_url`, `external_ids` — generic

### Vertical-spesifik UX

#### Vertical switcher header'da

```
[ATLAS logo]  ▾ Geophytes      ← clickable, açılır
              ─────────────────
              ✓ Geophytes
                Orchids (beta)
                Carnivorous plants (alpha)
              ─────────────────
                + Propose new vertical
```

Slack workspace switcher pattern. User'ın `last_active_vertical_id`
profile'da tutulur.

#### Sidebar bağlamı

Aktif vertical'a göre Commons bucket'ı değişir:
- Geophytes: Species + Metabolites + Publications + Researchers + Orgs
- Orchids: Species + Pollinators + Publications + Researchers + Orgs
  (Metabolites yerine Pollinators, çünkü orkide ekolojisinde primary topic o)
- Amphibians: Species + DiseaseSurveillance + Publications + Researchers + Orgs

#### Veri yalıtımı

Default query'ler vertical_id filter'lı:
```sql
SELECT * FROM species WHERE vertical_id = current_setting('app.vertical_id');
```
Bir RLS policy + GUC kombinasyonu ile bu otomatik enforce edilir.

#### Cross-vertical query

Power user "tüm vertical'lar arası endemik + threatened" sorgu
yapmak isterse: `?vertical=all` URL param ile bypass. Ana feed
default vertical-scoped.

### Permission model genişletme

```sql
CREATE TABLE vertical_maintainers (
  vertical_id text REFERENCES verticals(id),
  user_id uuid REFERENCES auth.users(id),
  role text CHECK (role IN ('maintainer','reviewer','contributor')),
  added_at timestamptz DEFAULT now(),
  PRIMARY KEY (vertical_id, user_id)
);
```

`is_admin()` → `is_vertical_maintainer(vertical_id, user_id)` çağrısı.
Şu anki tek global admin yerine her vertical'ın kendi maintainer'ı.

### Launch sequence (sıralı, atılgan değil)

1. **Şimdi**: Geophyte vertical'ında stabilize ol. Bu oturumda
   pek çoğu kapandı — IA fix + visual cleanup kalan iş.
2. **2-4 hafta**: `verticals` tablosu + `species.vertical_id` migration.
   Geophyte default, single-vertical UX hiç değişmiyor. **Sıfır UX risk**.
3. **4-8 hafta**: Vertical-aware RPC'ler. Filter pattern her listede
   benimsenir.
4. **8-12 hafta**: Vertical switcher UI (header'a). Sadece admin'lere açık.
5. **12-16 hafta**: İkinci vertical seç (önerim: **orkideler** — endemic
   + threatened + pollinator-conservation + ethnobotany örtüşmesi
   geofitlerle %70 ortak. IUCN orkide grubuyla anlaşman zaten var).
   Soft launch — beta watermark.
6. **16-24 hafta**: Üçüncü vertical (örn. carnivorous plants veya
   sukulent crassulaceae). Vertical abstraction olgunlaşır.
7. **24+ hafta**: Self-serve vertical creation (admin review ile).

### IUCN-uyumluluğu

Multi-taxon expansion IUCN'le **uyumsuz değil** — IUCN zaten her takson
için Red List yapıyor. Aksine, multi-taxon yaklaşımı GEOCON'u
"IUCN community of practice"'ten "IUCN community of communities"'e
çevirir. Bizim için pozitif konum.

**Risk**: Eğer commercialization-recognition bazı vertical'larda
(örn. amphibians için pet trade) sorunlu görünürse, **L5 katmanı
vertical config'inde flag'lenir** ("Commercialization recognition
disabled for this vertical").

## IX. Riskler ve dikkat noktaları

1. **Sidebar reorganization'ı bir kullanıcı testiyle doğrula** —
   senin gözünle ve 1-2 araştırmacıyla. Plan ileri sürüyor ama gerçek
   onboarding session'ı yapılmamış data point. ORCID Connect banner'a
   tıklayan kaç kullanıcı var, telemetry yok şu an.

2. **Telemetry eksikliği genel sorun**: Şu an hiçbir route view
   count'u, hiçbir click tracking yok. PostHog veya Plausible'ı
   önemli adımlarda mount etmek gerekir, aksi halde IA decisions
   yine intuition'a kalıyor.

3. **Multi-taxon expansion + IUCN re-application zamanlaması**:
   IUCN'e GEOCON'u yeniden başvuracaksak, başvuru anında "geophyte
   commons" pitch'i daha temiz. Multi-taxon expansion onaydan **sonra**
   yapılmalı. IUCN tarafı "scope creep" görmemeli.

4. **Verticals + IUCN-compatibility**: Senin not'unda "marketplace
   sofistike kaliyor" var. Multi-taxon expansion bu riski tetiklemez
   ama vertical-spesifik commercialization config'i (yukarıda VI.7
   risk note'unda yazdığım flag) gerekiyor.
