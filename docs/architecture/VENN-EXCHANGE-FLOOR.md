# Venn Exchange — THE FLOOR (living borsa floor, design)

> 4-agent design workflow. Chosen + fused concept. Build v1 next.

## Name
THE FLOOR — Venn Exchange

## One-liner
A trading floor seen from above where the tape, the sector heat and the live pulse are all driven by real, money-free counts — busy and alive at zero deals, honestly reading CURATING, because the atlas itself is the live feed.

## Floor layout
THE OVERVIEW (/exchange) BECOMES THE FLOOR — top to bottom, inside the existing ExchangeShell (keep its violet cosmos BG, AMBER gradient, sticky header). Two pieces live in different homes so the floor-feeling is everywhere but the heatmap stays on the Overview only:

SPINE (in ExchangeShell, sticky under the header, on ALL five layers): THE TAPE — a ~34px money-free ticker band, backdrop-blur over rgba(21,8,33,0.85). Left = a fixed, non-scrolling status nameplate (the CURATING pill + a LIVE heartbeat dot); center = the perpetual right-to-left marquee of real facts; right edge = a 24px fade so glyphs dissolve off-screen. Because it mounts in the shell it appears on Overview / Cambium / Board / Directory / Desk from one insertion — the persistent borsa spine.

BODY (Overview only, replacing the current two-Stat row), four stacked bands:

Band A — THE BOARD (opening-bell header). A dark-glass slab (rgba(28,12,44,0.55), 1px amber-18 border, inset top highlight so it reads as a machined panel). Left: the same status pill, larger, with honest copy 'CURATING · first listings forming'. Center: three monospace tote counters — 28 FUNDS MAPPED · 6 SECTORS · 47,052 SPECIES BACKING — that count up ONCE on mount (terminal boot), then hold. Right: a real UTC wall-clock ticking HH:MM:SS in monospace with an 'EST. 2026 · TALLINN' plate (the one genuinely-live ticking element — it is the time, never a price).

Band B — THE FLOOR (the centerpiece: a sector heatmap, the trading floor from above). A 6-row × 5-column grid: rows = the 6 fund verticals (biotech, materials, climate, biodiversity, cosmetics, agriculture), columns = the 5 funding stages (pre-seed, seed, series-a, growth, multi-stage), with a left axis-label column and a top axis header. Tile content (monospace): big fund count + a 3–4px sparkbar whose width = density + a micro-label of that cell's strongest thesis tag. A re-pivot toggle above the grid (By stage ⇄ By fund type) flips the 5 columns to the 5 fund kinds (vc/impact/corp_vc/strategic/foundation), re-racking heat from the same real arrays with a 350ms tween. Band C — THE EXITS. The three existing layer-entry cards (The Cambium / Deal Board / Directory) restyled as 'floor exits', plus a one-line provenance stamp: 'Heat = curated fund density (28 funds) × conservation backing. No prices. Verified 2026-06-09.'

Band D — WHO MEETS HERE + THE FIREWALL. The existing participants grid and the green firewall panel, copy unchanged (they already nail the serious tone).

## Ticker design
WHAT SCROLLS (the tape, looped seamlessly — render the segment array twice, translateX -50%, invisible seam). Reading left→right the rhythm alternates GREEN evidence and AMBER market, so the strip visibly pulses between conservation and commerce — the firewall as a heartbeat:

[ATLAS 47,052 spp ▲green] · [IUCN ASSESSED 664 ▲green] · [THREATENED · CR 52 EN 106 VU 203 ◆green] · [PUBLICATIONS 189 ▲green] · [CHAIN 282,312 facts ▲green] · [363 link types ◆green]
‖‖ ‹thin amber divider — 'the firewall'› ‖‖
[FUNDS 28 ◆amber] · [VC 15 ◆] · [IMPACT 6 ◆] · [CORP 3 ◆] · [STRATEGIC 3 ◆] · [FOUNDATION 1 ◆] · [STAGE seed·growth·series-a ◆amber] · [SECTOR biotech · materials · climate · biodiversity · cosmetics · agriculture ◆violet] · [DEALS 0 ‖muted] · [VENTURES 0 ‖muted]
‖‖ ‹wrap to status› ‖‖

THE GRAMMAR (the trick that lets it 'smell of commerce' while staying pure): each cell is LABEL · NUMERAL · GLYPH. The glyph is a typographic DIRECTION marker on a count, never a money movement: ▲ green #7BE3BE = an append-only / growing count (atlas, IUCN, publications, chain facts — true because those tables only ever grow); ◆ amber #F5A623 = a curated steady set (28 funds, the kind splits); ‖ muted #9b8c74 = flat / not-yet-open (DEALS 0, VENTURES 0 — reads as 'halted / pre-open', the borsa word for exactly our state, NOT a fake 0.00). No $, no %, no +/-N, no price column EVER. The sector strip (biotech · materials · climate…) is the real vertical_focus distribution and reads exactly like a Bloomberg index-sector strip ('TECH · ENERGY · FINANCIALS') — that single touch is most of the stock-exchange smell. The NUMBERS THEMSELVES NEVER TICK OR ANIMATE (faking a changing numeral would be a lie); liveness comes only from the perpetual scroll carrying real facts past you, the breathing status, and the color-coded direction glyphs.

THE RPC (one new money-free call, public, firewall-safe by construction):
public.get_exchange_tape() returns jsonb, language sql STABLE SECURITY DEFINER, set search_path = public, bridge, pg_temp, granted to anon + authenticated. Returns ONE payload:
{ status: 'curating'|'opening'|'open' (derived: opportunities=0 → curating; >0 → opening; outcomes>0 → open), as_of: <max last_verified date>, market: { opportunities, ventures, funds, fund_kinds: {vc:15,...} }, atlas: { species, iucn_assessed, threatened, publications, chain_facts, chain_types }, sectors: [{vertical,funds}…6], stages: [{stage,funds}…5], kinds: [{kind,funds}…5], grid_by_stage: [{vertical,stage,funds,top_thesis}…], grid_by_kind: [{vertical,kind,funds}…] }. The grid arrays are built with `cross join lateral unnest(vertical_focus)` × `cross join lateral unnest(stage_focus|kind)` over bridge.investors WHERE status='directory' (NOT zipped unnest — see build_spec note). Guarantee by construction: the SELECT list contains zero money/PII columns — it can only emit counts, labels and a status enum; bridge.opportunities is COUNTED, never projected. One RPC powers both the tape and the floor heatmap (the Overview fetches the full payload; inner layers use only status+market+atlas). 0-ERROR advisors is the bar; /crosscheck before ship since it touches the firewall.

## Motion system
PRIMITIVES (a deliberately small vocabulary — everything is transform/opacity/filter only, so no reflow, no jank):
1. CSS @keyframes marquee — the tape. One translate3d(0)→translate3d(-50%) on a doubled strip, will-change:transform, 60s linear infinite. Zero JS per frame; the compositor owns it.
2. CSS @keyframes pulse/breath — the status dot (reuse the EXISTING cmb-breathe keyframe from Cambium, 3s, do not add a new one), the LIVE blink (2s opacity 0.4↔1), and the Cambium live ring (already present).
3. requestAnimationFrame integer count-up — the three board tote counters + the floor heat tiles' boot-in. Runs ONCE ~700–900ms on mount, then cancels (no idle rAF). Re-fires once per real realtime delta.
4. CSS transition on background-color + a one-shot rAF number tween (350ms) — the re-pivot toggle (By stage ⇄ By fund type) 're-racks' the board.
5. One absolutely-positioned amber gradient bar, single translateX keyframe (~9s) — the scan line sweeping the heatmap like a radar/board refresh. Pure ambience, one GPU layer.
6. CSS transition transform/opacity (.15s) — every hover micro-interaction (tile lift scale(1.04), tape cell lift, layer-card border).
7. setInterval(1s) on one text node — the UTC clock (the only persistent JS tick; negligible).

WHAT MOVES & WHY: the tape scrolls (real facts stream past = 'the floor is busy'); the status dot breathes + LIVE blinks (the surface is live even though no deal is); the hot heat-tiles pulse ~±8% luminance over 4s with amplitude proportional to heat, COLD tiles barely move (density = heat, the literal brief) — crucially this is decorative oscillation around a FIXED value, the number never changes so nothing is faked; the scan line sweeps (board-refresh ambience); the clock ticks (real time, honest); on the first real opportunities 0→1 a single 600ms amber sweep crosses the tape once (the opening bell — silent, serious, event-driven, then removed; the only 'celebration', fired solely on a genuine state change).

THE LIVE PULSE (optional v1.1, real-event-driven): a useExchangeFloor hook subscribes to the ONE realtime-published, money-free, conservation-side table — program_tic_audit (confirmed in supabase_realtime). On a real tic INSERT: the PUBLICATIONS/VERIFIED counters re-count-up and one ripple crosses the floor. Subscription re-subscribes only on id change, refetch via ref, 600ms debounce tail — exactly the hardening pattern from commit a14097a0 / useProgramStream. No tic, no motion; silence is honest. No network polling otherwise.

PERFORMANCE BUDGET (no jank): hard cap — only opacity/transform/filter animate continuously; never animate width/top/left/box-shadow on a loop (glow is a static text-shadow). will-change:transform ONLY on the two persistent transforms (tape, scan line), never on all ~30 tiles (avoids layer explosion); stagger tile pulse delays so they never repaint on the same frame. Continuous JS cost ceiling = the 1s clock (and, in v1.1, an off-by-default IntersectionObserver on the realtime hook). Pause-when-hidden: animation-play-state:paused via IntersectionObserver + visibilitychange (the band is at the very top). Target locked 60fps, <2ms scripting/frame idle. SSR-safe: render labels with '—' placeholders (like the existing Stat) then hydrate — no layout shift.

prefers-reduced-motion: reduce → a true off-switch, not a dimmer, and the integrity fallback. One media-query block: marquee → static wrapping flex row (all facts still readable); count-ups → final values instantly; tile breathing + scan line + opening sweep → off (heat shown by static fill only); status dot static. Every real number, the CURATING status, and (in v1.1) live updates remain — motion is purely additive, never load-bearing for information.

## Borsa aesthetic
The single highest-leverage choice: every count, clock and axis numeral in fontFamily 'ui-monospace, Menlo, Consolas, monospace', fontVariantNumeric:'tabular-nums', letterSpacing 0.5, comma-grouped (47,052 · 282,312) — already the codebase convention in VennExchangeRoute/ExchangeBoard. This one choice does ~70% of the 'stock exchange' work. PALETTE reuses the established tokens only, no new colors: band background rgba(21,8,33,0.85) over the cosmos BG with backdropFilter blur(8–10px); hairlines rgba(245,166,35,0.16) (matches the header border); numerals cream #FFE6BC / #FFD15C; labels + sector words muted violet #9b86c9; conservation accent green #7BE3BE / #1D9E75. THE FIREWALL AS A COLOR LAW, visible on the surface: commerce numbers are amber, conservation backing is green; they sit side by side but NEVER blend into one value — green=evidence, amber=the market frame, and a literal '‖‖ firewall ‖‖' divider splits every tape loop. The status pill: borderRadius 99px, dark glass, a box-shadow 0 0 10px <color>66 glow ring on the dot, uppercase monospace label. Board chrome: 1px inset top-highlight (inset 0 1px 0 rgba(255,255,255,0.06)) so slabs read as machined panels, not flat divs. The heat ramp is the shared Cambium amber() so the floor and the stem cross-sections speak one palette. The Venn three-lobe glyph from Cambium reused as the floor's corner mark. No emojis anywhere in files (constraint #3). The overall reflex: a Bloomberg tape over a dark trading desk — but every glyph is a verified conservation or fund fact, and the honest word on the board is CURATING.

## Integrity & firewall
NO FABRICATED PRICE LINE (explicit, structural): there is not one $ figure, %, valuation, volume, up/down arrow, or invented traction anywhere on the surface. A real tape ticks prices because the market changes; ours don't, so we never animate a numeral. The borsa 'delta' glyphs (▲ ◆ ‖) are typographic direction markers on real COUNTS — ▲ = append-only/growing (true: those tables only grow), ◆ = curated steady set, ‖ = flat/pre-open — never a money movement. DEALS 0 and VENTURES 0 are shown as the borsa state 'halted / pre-open' (‖), not hidden and not a fake 0.00 — honest, even prestigious framing of a curated exchange that hasn't opened the doors. MONEY-FREE PUBLIC SURFACE BY CONSTRUCTION: all motion is fed only by money-free RPCs — the new get_exchange_tape() (counts + status enum; its SELECT list physically cannot emit a price, currency, ticket, contact, or opportunity-detail column; bridge.opportunities is counted, never projected), plus the existing get_exchange_public_stats and list_exchange_directory (already PII/money-stripped). The firewall is enforced in the FUNCTION BODY, not just the UI. CONSERVATION→COMMERCE IS ONE-DIRECTIONAL: species/IUCN/publication/chain counts only BACK sectors visually (green) and never carry a money value; no commerce figure ever writes back into a species page or conservation table — the atlas is merely cited, read-only. THE BRIDGE STAYS ADMIN-GATED: the richer bridge tables (ticket_min/max/currency, contact_email/name, warm_intro_path, provenance_snapshot) are NOT in the realtime publication and live behind the admin-only Desk; the only realtime channel the floor may subscribe to is program_tic_audit (conservation-side, money-free, already published). HONEST AT 0 AND AT 1: today the floor is genuinely busy — 28 real funds, a dense real heatmap, 47,052 real species, status CURATING with real copy ('first listings forming'), empty cells framed as 'open territory'. The day the founder declares the first outcome it flips to OPENING; the day the first bridge.opportunity opens it flips to OPEN and that sector tile gains a green live-listing ring — same RPC, no redesign, no fakery. The floor was honest at zero and becomes honest at one. /crosscheck (Codex) runs before ship because this touches the firewall; 0-ERROR advisors is the bar.

## Uniqueness
Most 'exchange' UIs fake liveness with random price jitter; this one is structurally honest and that IS the differentiator. The motion budget buys 'alive' while the data stays frozen-true: the tape scrolls REAL append-only conservation counts (47,052 species, 282,312 chain facts) so it reads like a Bloomberg index strip with zero invented numbers; the heatmap's heat is literally 'how much real evidence and how many real funds stand behind each sector', not price; the firewall is rendered as a visible color law (green evidence vs amber market never share a value, with a literal ‖‖ firewall divider). It is buildable today from data that already exists — a dense 6×5 real grid, 28 real funds, an honest CURATING bell — and it upgrades itself with no redesign: the same RPC that says CURATING at 0 says OPENING at 1 and OPEN at the first real listing, and (via the already-published program_tic_audit channel) the floor visibly reacts the moment the founder completes one real tic. The promise to the eye — 'this is a place where real things happen' — is never a lie, even, especially, at zero. It also reuses the codebase's own organs (Cambium's amber() ramp + cmb-breathe, the monospace stat convention, KIND_META colors, the a14097a0 realtime hardening), so the floor and the stem cross-sections feel like one organism, not a bolted-on widget.

## Real data feeds
- **The tape's market half (FUNDS 28, VC 15, IMPACT 6, CORP 3, STRATEGIC 3, FOUNDATION 1) and the Directory** ← bridge.investors WHERE status='directory' — 28 rows, via get_exchange_tape() / existing list_exchange_directory (PII+money stripped)
- **The sector strip (biotech 19 · materials 14 · climate 11 · biodiversity 9 · cosmetics 7 · agriculture 6) = the 6 heatmap ROWS** ← unnest(vertical_focus) over the 28 directory funds, grouped
- **The 5 heatmap COLUMNS (seed 9 · growth 8 · series-a 4 · multi-stage 4 · pre-seed 3)** ← unnest(stage_focus) over the 28 directory funds, grouped
- **Each heatmap TILE (fund density at vertical×stage, e.g. climate×seed=6, biotech×seed=6, materials×seed=6) and its top-thesis micro-label** ← bridge.investors CROSS JOIN LATERAL unnest(vertical_focus) × unnest(stage_focus) WHERE status='directory', grouped — real, dense, max cell 6
- **The re-pivot 'By fund type' columns (vc/impact/corp_vc/strategic/foundation)** ← unnest? no — kind is scalar: group by kind over directory funds (vc 15 / impact 6 / corp 3 / strategic 3 / foundation 1)
- **Tile / tooltip top-thesis tags (climate 10, bio-based-materials 8, biodiversity 8, biotech 7, regenerative 5, long honest tail)** ← unnest(thesis_tags) over the 28 directory funds, grouped — the strongest tag per grid cell
- **The tape's evidence half + the '47,052 SPECIES BACKING' board counter (ATLAS 47,052 spp ▲)** ← count(*) public.species
- **IUCN ASSESSED 664 and the THREATENED · CR 52 EN 106 VU 203 split (361 threatened)** ← count public.species where iucn_status in ('CR','EN','VU','NT','LC') = 664; threatened CR+EN+VU = 361; NT 79, LC 224
- **PUBLICATIONS 189** ← count(*) public.program_publications
- **CHAIN 282,312 facts · 363 link types** ← count public.chain_link_fact = 282,312; count public.chain_link_type = 363
- **'6 SECTORS' board counter / sector count** ← distinct vertical_focus over directory funds (6)
- **The market-status pill (CURATING today) + DEALS 0 ‖ + VENTURES 0 ‖** ← count bridge.opportunities = 0 and count public.commercialized_outcomes = 0 → status derives to 'curating'; flips to 'opening' at first opportunity, 'open' at first outcome
- **The UTC wall-clock HH:MM:SS** ← client Date() — genuinely live; it is the time, not a market figure
- **The live-pulse ripple + counter re-count (v1.1)** ← Supabase Realtime INSERT on public.program_tic_audit (confirmed published; conservation-side, money-free) — fires only on a real completed program tic
- **The 'Verified <date>' provenance stamp** ← max(last_verified) across bridge.investors directory rows (all 2026-06-09)
- **Active-vertical context (geophytes)** ← public.verticals — 1 public row (is_public=true); matches existing get_exchange_public_stats().verticals

## Open questions
- The 6×5 heatmap counts a fund in MULTIPLE cells (a multi-vertical, multi-stage fund lights several tiles), so column/row sums exceed 28. Is that the right read ('a fund is present in N sectors') or should we show each fund once? I recommend the multi-cell read with a 'tiles count presence, not funds' footnote — but it is a taste call.
- The market-status thresholds: CURATING at 0 opportunities, OPENING at 1+, OPEN at first outcome — or do you want OPEN to require a higher bar (e.g. 5+ live listings) so 'open' feels earned? Your call on the language too: CURATING / OPENING / OPEN, or Turkish equivalents.
- The live pulse (v1.1) ripples the floor on a real program TIC — but tics are conservation work, not commerce. Is that the right signal to animate on the commerce surface, or should the floor stay silent until a real bridge.opportunity/outcome lands (which means almost no motion-events for a while)? I lean: keep it, the atlas IS the live feed, but flag it as a firewall-taste decision for you.
- Should the Cambium also gain the event-driven ring-deposit / investor-ray draw-in animations from Direction 3 in this same pass, or is that a separate later ticket? It is real-data-safe (rings only grow on real completed stages) but expands the scope beyond the Overview floor.
- Do you want the tape on EVERY layer including the admin Desk, or should the Desk stay clean (it is the money-side workspace)? My default: tape on all public layers, suppressed on /exchange/desk.
