# Venn Exchange — cold-start playbook (the manual first move)

> The strategy (VENN-EXCHANGE-VC-STRATEGY.md) says: do the first real VC touch BY
> HAND before building any machine. The founder confirmed: warm-intro-only,
> native deal-room (later), the seed-fund list. This is the step-by-step.

## The honest bottleneck (data reality, 2026-06-09)
`commercialized_outcomes` is **EMPTY (0 rows)**. `bridge.opportunities` and
`bridge.investors` are empty too. So the Exchange has nothing to list — not
because VC access is hard, but because **no value output has been declared yet.**
The whole VC strategy sits on top of ONE real, evidenced commercialized outcome.
That is the true first move, and it is the founder's domain call (which real
result carries commercial value). Everything below is upstream of any VC.

## The chain (each step gates the next)
0. **Declare the FIRST commercialized outcome** — pick one real, evidenced value
   output of GEOCON work (a validated propagation protocol, a documented
   metabolite with application interest, a thesis/program result a buyer/investor
   would care about). Declare it on the conservation side (the CommercializedOutcomes
   panel / outcomes flow) — money-free, evidence-linked. Even a modest one is fine;
   it must be REAL and evidenced, never invented.
1. **Climb the verification ladder → `venn_verified`.** Only a venn_verified
   outcome can open a door (the firewall seam). This is the credibility the whole
   pitch rests on.
2. **Open the door** (admin, on the outcome) → an opportunity appears at
   `/exchange/desk`, with the evidence frozen.
3. **Seed the directory** — the ~15-20 funds below (the founder's confirmed list).
4. **Map warm paths** — for each fund, who can introduce you (ORCID co-authors,
   advisors, the conservation/biotech network).
5. **Draft the one-pager by hand** — the evidence box (template below).
6. **Send EXACTLY ONE warm double-opt-in intro** to the best-matched connector for
   the strongest-fit fund. Watch what happens. One careful touch at a time.

When that one human-run intro proves the artifact converts, the machine gets
built: native tokenized deal room + the directory import (P4, gated behind ≥1
real verified opportunity, firewall + /crosscheck on every step).

## Warm double-opt-in intro — email template (forwardable)
> To the CONNECTOR (not the VC):
>
> Subject: intro? — evidenced [vertical] value output, fits [Fund]'s thesis
>
> Hi [name], I am working with VENN Bioventures on a conservation-grounded value
> output that I think genuinely fits [Fund] — and I would value a warm intro if
> you are comfortable. The short of it: [one line — what the output is]. It is
> backed by a frozen, third-party-verifiable evidence record (IUCN status,
> verification ladder, [N] endorsements, DOI) — not a deck of projections. If it
> looks relevant, I can send a one-page, no-login evidence brief you can forward.
> No obligation — happy to drop it if the timing is off. — [you]

The connector forwards with one line; the VC opts in BEFORE the founder is
exposed. Only then does the one-pager link go out.

## One-pager — evidence box (the kill-shot, draft by hand)
Keep it to one page. The opening instrument; the full deck is held for after a bite.
```
[Species / output name]            [vertical]
THE ASK: [EKLE: kind + amount + use of funds]   — never a fabricated valuation
WHY NOW: [1-2 lines, real]
THE EVIDENCE (frozen, verifiable):
  • IUCN status: [tier]
  • Verification: venn_verified ([ladder])
  • Endorsements: [N]
  • Source: [DOI / outcome link]
THE COMMERCIAL THESIS: [EKLE: buyer segments, application] (no invented TAM)
TEAM: [public attribution only — names/ORCID, no private contact]
```
Rule: anything you cannot evidence stays `[EKLE:]`. Never invent traction/numbers.

## GDPR note (warm-intro-only keeps this light)
With warm double-opt-in intros only, there is no cold marketing email, so the
heavy ePrivacy/Art.14 obligations do not trigger. If/when a cold digest is ever
added, a dated Legitimate Interest Assessment + an Art.14 source-disclosure line
in every email become mandatory (VENN = Estonian controller). Not needed yet.

## What only the founder can do (the irreducible human acts)
- Decide which real result is the first commercialized outcome.
- Verify it (the judgment behind venn_verified).
- Map the real warm paths.
- Send the one intro, and take the call when a VC bites.
The machine does the rest (matching, drafting, link-minting, analytics) — once
there is one real opportunity to point it at.
